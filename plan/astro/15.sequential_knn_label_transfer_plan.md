# Plan: Sequential kNN Archetype Label Transfer (P56 → all younger postnatal ages)

## Context
Script 13 projects ages into P56 PCA space and only labels P56 cells. The user wants:
1. project each postnatal ages into P56 PCA space (n=50 PCs)
2. Sequential relay label transfer using kNN majority voting (not centroid NN):
   P56 → P28 → P14 → P7 → ... → earliest postnatal age

## Scripts to Write

### Script 15: `scripts/astro/15.sequential_knn_label_transfer.py`
Self-contained computation script. Does not depend on script 13 outputs.

**Algorithm**:
1. Load h5ad; extract all postnatal ages (Age starts with 'P'); no min-cell filter
2. HVG selection P56 cells (`select_hvg` from `scripts/common.py`)
3. Joint normalization (`norm` from `scomics.utils`) on HVG columns for all cells; depths = `x.sum(axis=1)` (computed before HVG subsetting)
4. Fit `pca_project` (sklearn PCA, N_DIMS_KNN=20 components) on P56 cells only; project all postnatal-age cells through `pca_project`; drop same PCs as PCHA step (`DROP_PCS`) so kNN transfer space matches PCHA space
5. **P56 archetype assignment**:
   - Create `SCA(x_p56_norm_hvg, donors_p56)`; call `sca.setup_feature_matrix(method='data')`
   - Run `run_noc_sweep` (from `scripts/common.py`) on P56 cells (NDIM=5, NOC=4, DROP_PCS=[1]) → this internally fits a separate `pca_pcha` (6 components, drop PC2) via `SCA.proj_and_pcha`
   - Assign each P56 cell to nearest archetype vertex in PCHA space (same logic as script 13 lines 80–83)
6. **Sort ages oldest → youngest** by parsing numeric value after 'P' (e.g. P56=56, P28=28, ...)
7. **Sequential kNN relay**:
   - Start: P56 cells are labeled (archetype labels from step 5)
   - For each step `older_age → younger_age`:
     - Fit `KNeighborsClassifier(n_neighbors=K_NEIGHBORS)` on older-age cells' `pca_project` coordinates (N_DIMS_KNN − len(DROP_PCS) columns after PC drop)
     - Predict labels for all younger-age cells via majority vote
     - Younger-age cells become new reference for next step
8. Collect per-age DataFrames; save per-age and combined parquets

**File path constants** (top of script, capitalized):
```python
INPUT_FILE            = .../local_data/source/gao25/DevVIS_scRNA_AstroTE.h5ad
RES_DIR               = .../local_data/res/astro
PARQUET_AGE_TEMPLATE  = RES_DIR / '15.labels_{age}.parquet'
PARQUET_ALL           = RES_DIR / '15.labels_all_ages.parquet'
```

**Key parameters** (top of script, capitalized):
```python
NDIM           = 5       # PCHA dims (same as script 13)
NOC            = 4       # archetypes
NREPEATS       = 10
N_TOP_GENES    = 2000
DROP_PCS       = [1]     # drop PC2 in PCHA space
N_DIMS_KNN     = 20      # joint PCA components; used for kNN distance and visualization
K_NEIGHBORS    = 15      # kNN k
```

**Reused utilities**:
- `scripts/common.py`: `select_hvg`, `run_noc_sweep`
- `src/scomics/main.py`: `SCA`
- `src/scomics/utils.py`: `norm`
- `sklearn.decomposition.PCA` (`pca_project`, N_DIMS_KNN=20 components — separate from the internal PCHA PCA)
- `sklearn.neighbors.KNeighborsClassifier` (kNN transfer)

**Output columns per parquet**: `PC1, PC3, PC4, …` (from `pca_project`, with DROP_PCS columns removed), `donor_name`, `age`, `archetype`

### Script 16: `scripts/astro/16.sequential_knn_label_transfer_viz.py`
Visualization of transferred labels. Loads script-15 combined parquet and renders scatter plots and an abundance barplot.

**Inputs**: `15.labels_all_ages.parquet`

**Visualizations** (follow patterns from scripts 02 and 14):
- 4-subplot scatter for weekly timepoints only (`SCATTER_AGES = ['P0', 'P7', 'P14', 'P21', 'P28', 'P56']`), colored by archetype label: row 1 has 3 2D panels (PC1 vs PC3, PC1 vs PC4, PC3 vs PC4 via `PANELS`); row 2 has a full-width 3D panel (PC1–PC3–PC4 via `PANEL_3D`). PC2 already absent from parquet — no additional dropping in script 16.
- Single HTML with all ages colored by archetype or age (same 4-subplot layout)
- Stacked barplot (one bar per age, all 25 postnatal ages in natsorted order) showing archetype fraction — mirrors script 02's `stacked_bar_html` pattern

**Imports**: `from viz import scatter_categorical_html, stacked_bar_html`

**Note on `scatter_categorical_html`**: accepts `panels=[(col_x, col_y, xlabel, ylabel), ...]` and `panel_3d=(col_x, col_y, col_z, xlabel, ylabel, zlabel)`. When `panels` is provided, renders N 2D subplots on row 1 (width = 550 × N). When `panel_3d` is also provided, appends a 3D subplot on row 2 spanning all columns (height = 1100).

**File path constants**:
```python

```

**Key parameters**:
